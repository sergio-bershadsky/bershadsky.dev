## The Morning Briefing

The session-start hook provides automatic context:

```markdown
# Session Start Briefing Hook

When a session starts, provide context automatically.

## Checks to Run

1. Team Activity
   - Messages since last session
   - Decisions that need attention
   - Blockers mentioned

2. Your Tasks
   - Sprint items assigned to you
   - Pending reviews
   - Overdue action items

3. System Health
   - Stale documentation warnings
   - Undocumented meetings
   - Pending ADRs

## Output Format

Keep it brief—one line per category.
Only show categories with items.
Use counts, not full lists.
```

Here's what it looks like in practice:

```
You:    [Opens Claude]

Claude: Good morning! Here's your context:

        - Overnight: 5 messages in #engineering
        - Sprint: 3 items in progress, 2 in review
        - Pending: 1 meeting to document (yesterday's sync)
        - Stale: API guide needs review (45 days)

        Quick actions:
        - /recap — Full overnight summary
        - /standup — Prepare your update
        - /transcribe — Document yesterday's meeting
```

The psychology: start every session with context, not confusion. Know where you are before you start moving.

## Standup Automation

The standup skill eliminates the daily writing task:

```markdown
# Standup Skill

Generate a complete standup update automatically.

## Procedure

### Step 1: Gather Yesterday's Activity
Check:
- Git commits you made
- PRs merged or reviewed
- Issues moved in task tracker
- Meetings attended (from tracking)

### Step 2: Identify Today's Focus
From task tracker:
- Issues assigned to you
- Items marked "In Progress"
- Upcoming deadlines

### Step 3: Find Blockers
Search for:
- Issues with "blocked" label
- Dependencies waiting on others
- Open questions from meetings

### Step 4: Format Update
Structure:
- Yesterday: 2-3 bullets
- Today: 2-3 bullets
- Blockers: Only if relevant

Keep each bullet under 15 words.
```

Example output:

```
You:    /standup

Claude: ## Standup — January 15

        **Yesterday:**
        - Merged Redis caching PR (#234)
        - Reviewed Alice's auth changes
        - Attended Q1 planning meeting

        **Today:**
        - Implement cache invalidation logic
        - Write tests for caching layer
        - 2pm: Design review meeting

        **Blockers:**
        - Waiting on API specs from Bob

        ---
        Ready to share? [Copy to clipboard]
```

Automation options range from manual (/standup whenever you need it) to scheduled (hook at 8:55am generates draft) to smart (session start detects it's standup time).

## The Feedback Loop

Your second brain improves over time:

```
FEEDBACK LOOP
─────────────────────────────────────────────────────────
USE IT ────▶ NOTICE GAPS ────▶ ADJUST
  │               │               │
  ▼               ▼               ▼
Run /standup  "Missing PR     Add PR check
              info"           to skill
```

Common adjustments:

- Morning briefing too long? Add priority filtering
- Missing team messages? Expand channel list
- Standup missing recent work? Add git integration
- Too many stale warnings? Adjust threshold

Questions to ask weekly:

- What did I have to look up manually?
- What context was missing when I needed it?
- What repetitive task did I do by hand?
- What would have helped catch an issue earlier?
