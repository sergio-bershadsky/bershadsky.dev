# Teaching Your Brain: Skills

> Your second brain understands concepts. Now let's teach it procedures — repeatable workflows it can execute.

**Reading time:** 7-10 minutes | **Audience:** Semi-Technical | **Words:** ~1,600

---

## The Hook

> "Claude answers questions well. But what about tasks? The things you do every week, the same way, wishing you could just... automate them?"

Skills are the bridge from assistant to automation. One command, many actions, zero thinking required.

---

## What Are Skills?

**The simple explanation:**
> Skills are reusable procedures that Claude follows when you trigger them. Think of them as "macros for knowledge work."

**Examples:**
```
/weekly-recap     → Generates your weekly summary
/document-meeting → Processes transcript into notes
/standup          → Prepares your daily update
/catchup          → Summarizes what you missed
```

**What makes skills different from just asking:**
```
WITHOUT SKILLS                    WITH SKILLS
──────────────                    ───────────
"Can you summarize               /recap
 the week's activity
 including team chat,
 issues closed, and
 decisions made? Format
 it like last time..."
```

**Skills are:**
- Written in plain language (not code)
- Stored in your second brain
- Triggered with a command
- Consistent every time

> Skills are like teaching Claude your secret recipes. Once taught, it can make the dish exactly the same way every time — no more "I think it was two tablespoons?" moments.

## Anatomy of a Skill

**Core structure:**

```
SKILL: /daily-recap

WHEN: User asks for daily summary

DO:
1. Check team chat (last 24 hrs)
2. Review task tracker updates
3. Summarize decisions made
4. List action items

OUTPUT: Formatted summary
```

**Skill file structure:**

```markdown
# Daily Recap Skill

Generate a summary of the last 24 hours of activity.

## When to Use

- Morning catchup
- Before team meetings
- When returning from time off

## Procedure

### Step 1: Gather Team Activity
Check the team chat for messages in the last 24 hours.
Focus on: decisions, blockers, questions asked.

### Step 2: Review Task Progress
Check the task tracker for:
- Issues closed
- Issues moved to review
- New blockers identified

### Step 3: Check for Decisions
Search discussions for any decisions made.
Link to formal decision records if they exist.

### Step 4: Generate Summary
Format the output as:
- **Team Highlights** (2-3 bullets)
- **Sprint Progress** (what moved)
- **Decisions Made** (if any)
- **Action Items** (for you)
```

**The key insight:**
> Skills are instructions, not code. You're writing what Claude should do in plain language. Claude figures out the how.

## Building Your First Skill

**Choose your first skill:**

Good candidates:
```
REPETITIVE + STRUCTURED = GOOD SKILL
─────────────────────────────────────
Weekly reporting           ✓
Meeting preparation        ✓
Code review checklist      ✓
Status update generation   ✓
One-off investigations     ✗
Creative writing           ✗
```

**Step-by-step creation:**

```
1. Identify a task you do repeatedly
   ↓
2. Write out the steps (as if teaching someone)
   ↓
3. Define the expected output format
   ↓
4. Create the skill file
   ↓
5. Test and refine
```

**Example: Creating a /standup skill:**

```markdown
# Standup Skill

Generate a standup update for team meetings.

## Procedure

### Step 1: Review Yesterday's Work
Check my commits and merged PRs from yesterday.
Check task tracker for issues I moved.

### Step 2: Identify Today's Focus
Look at issues assigned to me in "In Progress."
Note any meetings on the calendar.

### Step 3: Flag Blockers
Check for issues I'm waiting on.
Note any dependencies.

### Step 4: Format Update
Structure as:
- **Yesterday:** What I completed
- **Today:** What I'm working on
- **Blockers:** What's in my way

Keep each section to 2-3 bullets maximum.
```

## Skill Design Patterns

**Pattern: Information Gathering**
```
Purpose: Collect and synthesize data from multiple sources
Example: /catchup, /sprint-status
Flow:   Query → Aggregate → Summarize
```

**Pattern: Content Generation**
```
Purpose: Create structured output from context
Example: /standup, /weekly-report
Flow:   Gather → Format → Present
```

**Pattern: Analysis**
```
Purpose: Evaluate and assess something
Example: /review-pr, /audit-dependencies
Flow:   Read → Analyze → Report
```

**Pattern: Workflow Automation**
```
Purpose: Execute a multi-step process
Example: /release-notes, /document-meeting
Flow:   Trigger → Steps → Complete
```

**Diagram:**
```
SKILL PATTERNS
────────────┬────────────┬────────────┬──────────
  GATHER    │  GENERATE  │  ANALYZE   │ AUTOMATE
            │            │            │
  /catchup  │  /standup  │  /review   │ /release
  /status   │  /report   │  /audit    │ /deploy
```

## When Skills Shine

**Perfect for skills:**
- Consistent structure every time
- Multi-step processes
- Synthesizing from multiple sources
- Things you'd write a script for (but don't want to)

**Not ideal for skills:**
- One-off creative tasks
- Highly variable situations
- Things requiring human judgment throughout

**The skill test:**
> Ask yourself: "If I write out these steps, could someone follow them mechanically?" If yes, it's a skill. If no, it's a conversation.
