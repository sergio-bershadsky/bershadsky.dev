# Connecting Your Tools (MCP)

> Your second brain is smart, but it's been living in isolation. Time to introduce it to your other tools.

Your AI knows what you've documented. But it has no idea what your team discussed yesterday, what's blocking the sprint, or that meeting you missed. Let's fix that.

A second brain with only local knowledge is like a brilliant consultant who refuses to check email. Smart, but missing critical context.

## What Is MCP?

**The simple explanation:** MCP (Model Context Protocol) is how Claude talks to other services. Think of it as giving Claude a phone to call your other apps.

**The slightly more technical explanation:** MCP servers are small programs that translate between Claude and external services. They handle authentication, data formatting, and all the boring stuff so you don't have to.

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Claude     │────▶│  MCP Server  │────▶│   Service    │
│   (Brain)    │     │  (Translator)│     │  (Slack/etc) │
└──────────────┘     └──────────────┘     └──────────────┘
```

**What this means for you:**
- Claude can read your team messages
- Claude can check your task tracker
- Claude can pull meeting transcripts
- All without you copy-pasting things around

> MCP sounds like a government acronym for something classified. It's actually just "let Claude read your Slack" but make it enterprise.

## The Connection Types

Common MCP connections give Claude different superpowers:

```
CONNECTION TYPE          WHAT CLAUDE CAN DO
───────────────          ──────────────────
Team Chat                Read channel history
                         Summarize discussions
                         Find specific messages

Task Tracker             See sprint status
                         Check who's blocked
                         Review issue details

Meeting Transcription    Import transcripts
                         Extract action items
                         Find past discussions

Calendar                 Check availability
                         See upcoming meetings
                         Context about your day
```

**How connections work in practice:**

```
You:    "What did the team discuss about the API redesign?"

Claude: [via MCP] → Searches team chat
        [via MCP] → Finds relevant threads
        [via local docs] → Checks for related decisions

Result: "Based on last week's discussion in #engineering,
         the team decided to prioritize the v2 endpoints.
         Here's the summary..."
```

> Each connection alone is useful. Combined, they're transformative. Claude can now answer: "What's blocking the sprint, who discussed it, and what did we decide last time this happened?"

## Setting Up Your First Connection

**Recommended first connection: Team Chat**

Why team chat first:
- Highest context value
- Immediate usefulness
- Easy to verify it's working

**Generic setup process:**

```
Step 1: Find or create the MCP server
        ↓
Step 2: Configure authentication
        ↓
Step 3: Tell Claude about the connection
        ↓
Step 4: Test with a simple query
```

**Testing the connection:**

```
You:    "Can you check what the team discussed in #general today?"

Claude: [If working] "Here's a summary of today's discussion..."
        [If not working] "I don't have access to that channel..."
```

**Troubleshooting checklist:**
- Server is running
- Authentication token is valid
- Permissions are granted
- Channel/resource exists
- Claude can see the server in available tools

## Security & Privacy Notes

**What data flows where:**

```
YOUR TOOLS ──────▶ MCP SERVER ──────▶ CLAUDE
                        │
                   Data stays local
                   (usually)
```

**Key principles:**

1. **Minimal permissions:** Only grant access to what Claude needs
2. **Review scopes:** Understand what each permission allows
3. **Sensitive data:** Some things shouldn't flow to any AI
4. **Team awareness:** Make sure colleagues know Claude can read channels

**What NOT to connect (probably):**
- HR or legal channels
- Salary discussions
- Personal DMs (unless you really want to)
- Channels with sensitive customer data

> "With great power comes great responsibility." — Someone, probably about API permissions.
