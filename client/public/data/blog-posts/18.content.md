# The Opening

> "You forgot to update the documentation again. The tests ran but you didn't check the results. The file you created isn't in the navigation. Again."

We all have blind spots. Hooks are the friendly nudge that catches them before they become problems.

---

## What Are Hooks?

**The simple explanation:**

> Hooks are automatic triggers that fire when certain events happen. They watch for situations and respond without you asking.

**The key difference:**

```
SKILLS                        HOOKS
──────                        ─────
You trigger them              They trigger themselves
"Do this now"                 "When X happens, do Y"
On-demand                     Automatic
Pull                          Push
```

**Real-world analogy:**

> A skill is like ordering coffee. A hook is like the coffee maker that starts automatically when your alarm goes off.

**What hooks can do:**

- Remind you about things you forget
- Block dangerous operations
- Add context at the right moment
- Enforce consistency

> Hooks are like having a responsible adult looking over your shoulder. Except this adult never gets tired, never judges you, and can be turned off when you're feeling rebellious.

---

## Hook Events

**When hooks can fire:**

```
EVENT                    WHEN IT HAPPENS
─────                    ───────────────
Session Start            You begin a new conversation
Before Tool Use          Before Claude uses a tool
After Tool Use           After a tool completes
Session End              You're about to close
File Created             A new file is written
File Modified            An existing file changes
```

**Visual timeline:**

```
SESSION START ──────▶ "Good morning! You have 3 pending reviews."

   │
   ▼
WORKING... ─────────▶ [Normal conversation]
   │
   ▼
BEFORE TOOL USE ────▶ "Wait, that command looks destructive..."
   │
   ▼
TOOL EXECUTES ──────▶ [Action happens]
   │
   ▼
AFTER TOOL USE ─────▶ "That file isn't in the sidebar yet."
   │
   ▼
SESSION END ────────▶ "You have uncommitted changes."
```

**The most useful hooks:**

1. **Session Start:** Morning briefing, context loading
2. **After File Write:** Navigation checks, format validation
3. **Session End:** Uncommitted work reminders
4. **Before Tool Use:** Safety checks (careful with this one)

## Practical Hook Examples

**Example 1: Freshness Check (Session Start)**

```
TRIGGER: Session starts
CHECK:   Is documentation older than 30 days?
ACTION:  Warn about stale sections

Output:
"Heads up: The API guide hasn't been reviewed in 45 days.
 Consider updating it soon."
```

**Example 2: Sidebar Check (After File Write)**

```
TRIGGER: New .md file created
CHECK:   Is file in navigation config?
ACTION:  Warn if missing

Output:
"New file detected: docs/guides/new-feature.md
 This file isn't in the sidebar yet. Add it?"
```

**Example 3: Uncommitted Work (Session End)**

```
TRIGGER: Session ending
CHECK:   Any uncommitted changes?
ACTION:  Remind before closing

Output:
"You have 3 modified files that aren't committed:
 - docs/guides/setup.md
 - CLAUDE.md
 - skills/recap/SKILL.md

 Commit before closing?"
```

**Example 4: Safety Net (Before Tool Use)**

```
TRIGGER: Destructive command attempted
CHECK:   Does command match dangerous pattern?
ACTION:  Block or warn

Output:
"That command would delete the entire directory.
 Are you sure? [Confirm / Cancel]"
```

**How hooks work:**

```
                    ┌─────────────────┐
EVENT HAPPENS ─────▶│  HOOK CHECKS    │
                    │  condition      │
                    └────────┬────────┘
                             │
              ┌──────────────┴──────────────┐
              ▼                              ▼
        CONDITION MET               CONDITION NOT MET
              │                              │
              ▼                              ▼
      TAKE ACTION                     DO NOTHING
   (warn, block, add)             (silent continue)
```

## Building Your First Hook

**Choose a good first hook:**

```
GOOD FIRST HOOKS              NOT YET
────────────────              ───────
Session start reminder        Complex multi-step checks
Simple file checks            Blocking dangerous commands
Stale content warnings        Authentication validation
```

**Creating a session start hook:**

```python
# hooks/session-start.py

def on_session_start():
    """Check for things worth mentioning at session start."""

    reminders = []

    # Check for stale documentation
    stale_docs = find_stale_docs(days=30)
    if stale_docs:
        reminders.append(f"Stale docs: {len(stale_docs)} files")

    # Check for pending reviews
    pending = get_pending_reviews()
    if pending:
        reminders.append(f"Pending reviews: {len(pending)}")

    # Return reminders to show
    return reminders
```

**Alternatively, prompt-based hooks (no code):**

```markdown
# Session Start Hook

When the session starts, check:

1. Are any documentation sections older than 30 days?
2. Are there pending decision records?
3. Are there undocumented meetings?

If any issues found, mention them briefly.
Keep it to one line per issue.
Don't block — just inform.
```

**Placement:**

```
your-second-brain/
├── hooks/
│   ├── session-start.md     # or .py
│   ├── sidebar-check.md
│   └── README.md
```

## Hook Philosophy

**The golden rules:**

1. **Helpful, not annoying**
   - Inform, don't lecture
   - One-liners, not paragraphs
   - Easy to dismiss

2. **Warnings over blocks**
   - Blocking should be rare
   - Trust the user's judgment
   - Make overriding easy

3. **Silence is golden**
   - Only speak when something's wrong
   - No "all checks passed!" noise
   - Invisible when not needed

**The hook spectrum:**

```
SILENT ──────────────────────────────────▶ BLOCKING
   │                                           │
   │  Inform    Warn    Suggest    Require     │
   │                                           │
   ▼                                           ▼
"FYI: stale doc"         "Cannot delete main branch"
```

**Anti-patterns:**

- Hooks that fire constantly
- Blocking without clear reason
- Long explanations
- No way to override

> A well-designed hook is like a good friend who reminds you about your ex's birthday before you accidentally like their Instagram post. Subtle. Preventative. Life-saving.
